### 1.1 核心技术架构

#### 1.1.1 框架选择与技术栈分层

**工具链层**:

- **Webpack 5**: Next.js内置模块打包与代码分割

### 2.1 Framer Motion 动画系统架构

#### 2.1.1 进入退出动画模式

```typescript

```

#### 2.1.2 物理弹簧动画参数体系

| 弹性程度 | bounce值 | stiffness值 | damping值 | 应用场景 |
|---------|---------|------------|-----------|---------|
| 轻微弹性 | 0.3 | 200 | 25 | 按钮悬浮 |
| 标准弹性 | 0.4 | 180 | 20 | Tab切换 |
| 强烈弹性 | 0.5 | 150 | 15 | 弹窗出现 |
| 果冻弹性 | 0.6 | 120 | 10 | 彩蛋触发 |

**实例 - 修炼事件弹窗**:
```typescript

// 产生类似果冻晃动的欢快效果
```

#### 2.1.3 AnimatePresence 模态管理

```typescript
<AnimatePresence mode="wait">
  {activeSystem === 'aura' && (
    <motion.div
      key="aura"
      initial={{ opacity: 0, x: 20 }}
      animate={{ opacity: 1, x: 0 }}
      exit={{ opacity: 0, x: -20 }}
      transition={{ duration: 0.4 }}
    />
  )}
</AnimatePresence>
```

**mode="wait" 关键作用**: 确保旧组件完全退出后新组件才进入，避免视觉重叠和布局抖动

## 5. 特效实现方式与视觉呈现效果

### 5.1 Canvas 粒子系统架构详解

#### 5.1.1 标准粒子生命周期

```
出生 → 匀速运动 + 随机扰动 → 生命衰减 → 尺寸渐变透明 → 死亡过滤
    ↓        ↓           ↓           ↓
createRipple → 物理更新 → age++ → size*life → return life>0
```

#### 5.1.2 边界安全防护机制

**双重防线防止崩溃：**

1. **提前过滤防线**：在循环开始时移除濒死粒子
   ```typescript
   p.life -= decay
   if (p.life <= 0.01) return false
   ```

2. **下限截断防线**：所有渲染参数设置最小值
   ```typescript
   const size = Math.max(0.1, p.size * p.life)
   const alpha = Math.max(0, p.life / maxLife)
   ```

### 5.2 七大特效系统详解

#### 5.2.1 鼠标轨迹特效 - MouseTrailEffect

**技术实现：**
- 运动差分检测：只在鼠标移动时生成新粒子
- 粒子池容量：80个活动粒子上限
- 重力模拟：每帧 vy -= 0.02，模拟微微上扬
- 空气阻力：vx *= 0.98，模拟空气摩擦

**视觉效果：**
- 彩虹色相循环：`hsla(${p.hue}, 70%, 60%, ${alpha})`
- 双层光晕结构：实体核心 + 外围扩散晕
- 渐隐轨迹：随生命周期自然消散

#### 5.2.2 时光长河 - TimeRiverCanvas

**3D透视粒子系统：**

```typescript
const perspective = 100 / (100 + p.z)
const size = p.size * perspective * p.life
```

**视觉层次：**
- Z轴深度变化模拟3D空间
- 正弦函数扰动：`sin(time + p.z * 0.1)`产生流动感
- 时代色彩渐变：从远古暖色调到未来冷色调
- 鼠标斥力场：靠近粒子产生排斥推开效果

#### 5.2.3 因果蝴蝶效应 - KarmaButterflyEffect

**涟漪连锁机制：**
```typescript
// 业力连锁，一次点击产生strength*2次级涟漪
for (let i = 0; i < strength * 2; i++) {
  setTimeout(() => {
    createRipple(x + randomOffset, y + randomOffset)
  }, i * 150)  // 每150ms扩散一圈
}
```

**视觉表现：**
- 善恶双色系统：绿色善业涟漪 / 红色恶业涟漪
- 放射状粒子爆炸：点击中心8方向粒子喷射
- 中心🦋图腾：永恒呼吸动画

#### 5.2.4 灵气潮汐系统 - AuraTideSystem

**纯DOM动画实现：**
- 20个悬浮粒子统一相位错开
- 中央发光径向渐变：`radial-gradient(at center, 主题色)`
- 修炼事件弹窗：Spring弹性动画弹出

### 5.3 内存管理策略

## 9. 各模块间的关联与协作机制

### 9.1 数据层共享机制

#### 9.1.1 公共类型系统
- `src/types/common.ts` 统一定义五行、十二时辰、粒子等基础结构
- 所有7个特效组件共享同一套结构定义
- 修改一处，全局生效

#### 9.1.2 工具函数层
- Canvas清理、尺寸调整、粒子绘制等通用操作
- 性能Hooks统一提供防抖节流
- 客户端渲染统一解决方案

### 9.2 组件组合机制

#### 总览页组件注入模式

```typescript
const ComponentMap = {
  AuraTideSystem,
  JuneFortunePanel,
  DragonBoatFestival,
  TimeRiverCanvas,
  SixPathsWheel,
  KarmaButterflyEffect,
  ProphecyTimeline,
}

// 根据activeSystem动态渲染
const ActiveComponent = ComponentMap[activeSystem]
<ActiveComponent />
```

**优势：**
- 无props透传问题
- 组件完全独立，可单独使用也可组合
- 新增系统只需加入ComponentMap即可

### 9.3 状态隔离原则

**每个特效组件100%自治：**
- 独立useState管理
- 独立useEffect生命周期
- 独立ref管理Canvas资源
- 组件间无任何直接依赖

**松耦合优势：**
- 删除任何一个系统不影响其他
- 可以单独在任意页面嵌入任意组件
- 新增系统不需要修改既有代码
